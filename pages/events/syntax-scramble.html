Event Description

Syntax Scramble is an intense coding challenge that tests your programming skills without the safety net of compilers or execution. Write code blindly, debug others' work, and see if your solutions stand up to scrutiny in this multi-round competition.

This event will push your programming knowledge to the limits, requiring a deep understanding of syntax, logic, and debugging techniques. Do you have what it takes to code without seeing the output?

Competition Format

1

Round 1: Blind Coding (45 minutes)

Participants will be given problem statements and must write code without execution or compilation. You must rely solely on your knowledge of syntax and logic to create a working solution.

2

Round 2: Debugging Phase (25 minutes)

Solutions will be swapped between participants. Your task is to identify and fix any errors in another participant's code, still without execution.

3

Round 3: Execution & Judging (20 minutes)

Finally, the debugged code will be executed. Points will be awarded based on accuracy, efficiency, debugging skill, and creativity in problem-solving.

Evaluation Criteria

40% Accuracy

Does the output match the expected result?

30% Efficiency

Is the code optimized?

20% Debugging Skill

How well the participant corrected errors

10% Creativity

Unique or clever fixes

Grounds for Disqualification

  • Using online compilers
  • Copying code from other participants
  • Any other violation of the competition rules
